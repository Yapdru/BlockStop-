API Documentation

Complete reference for BlockStop PRO REST API

Overview

The BlockStop PRO API provides programmatic access to email and file security analysis. Build custom applications, integrate with existing tools, and automate your security workflows.

API Access: Available on PRO and MAX tiers

Authentication

API Keys

All API requests require authentication using an API key. Generate your API key in Settings → API Keys.

Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY
Security: Keep your API key secret. Never commit it to version control. Regenerate keys if compromised.

Base URL

https://api.blockstop.io/v1

Email Analysis

POST /email/analyze

Analyze an email for threats including phishing, malicious links, and spam indicators.

Request Parameters

Parameter Type Description
email_content * string Full email content including headers (MIME format)
sender_email opt string Sender's email address for reputation check
include_headers opt boolean Parse and analyze email headers (default: true)
sensitivity opt string Detection sensitivity: low, medium, high (default: medium)

Request Example

curl -X POST https://api.blockstop.io/v1/email/analyze \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "email_content": "From: sender@example.com\nTo: user@company.com\n...",
    "sender_email": "sender@example.com",
    "sensitivity": "high"
  }'

Response

200 OK

File Scanning

POST /file/scan

Scan a file for malware, trojans, ransomware, and other threats.

Request Parameters

Parameter Type Description
file * binary File to scan (multipart/form-data). Max 500MB
filename opt string Original filename for analysis
sensitivity opt string Detection sensitivity: low, medium, high (default: medium)

Request Example

curl -X POST https://api.blockstop.io/v1/file/scan \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@/path/to/file.exe" \
  -F "sensitivity=high"

Scan History

GET /scans

Retrieve your scan history with filtering and pagination.

Query Parameters

Parameter Type Description
limit opt integer Number of results (default: 20, max: 100)
offset opt integer Number of results to skip (default: 0)
type opt string Filter by type: email, file (default: all)
threat_level opt string Filter by threat: safe, suspicious, dangerous

Response Example

200 OK

{
  "status": "success",
  "scans": [
    {
      "scan_id": "scan_abc123",
      "type": "email",
      "risk_score": 85,
      "timestamp": "2024-06-21T10:35:00Z"

      "type": "file",
      "filename": "document.pdf",

Error Handling

The API returns standard HTTP status codes. Error responses include a detailed error message:

Error Response Format

{
  "status": "error",
  "error_code": "INVALID_REQUEST",
  "message": "Email content is required",
  "details": {
    "missing_fields": ["email_content"]
  }
}

Common Error Codes

  • 400 Bad Request: Invalid request parameters
  • 401 Unauthorized: Invalid or missing API key
  • 403 Forbidden: Insufficient permissions for this operation
  • 429 Too Many Requests: Rate limit exceeded
  • 500 Internal Server Error: Server error during processing

Rate Limiting

Tier Requests/Minute Concurrent Scans
PRO 60 5
MAX 300 25
Rate Limit Headers: Check response headers for rate limit information:
X-RateLimit-Limit: 60 X-RateLimit-Remaining: 45 X-RateLimit-Reset: 1624267800

Code Examples

Python

import requests

api_key = "YOUR_API_KEY"
headers = {"Authorization": f"Bearer {api_key}"}

# Email analysis
response = requests.post(
    "https://api.blockstop.io/v1/email/analyze",
    headers=headers,
    json={"email_content": "..."}
)
print(response.json())

# File scanning
with open("file.exe", "rb") as f:
    files = {"file": f}
    response = requests.post(
        "https://api.blockstop.io/v1/file/scan",
        headers=headers,
        files=files
    )
print(response.json())

JavaScript/Node.js

const apiKey = "YOUR_API_KEY";

// Email analysis
fetch("https://api.blockstop.io/v1/email/analyze", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${apiKey}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    email_content: "..."
  })
}).then(r => r.json()).then(console.log);
