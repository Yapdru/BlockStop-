================================================================================
                          MAINTENANCE GUIDELINES
================================================================================

For Developers:
  • Always use semantic HTML first
  • Import a11y utilities for common patterns
  • Add ARIA labels to icon-only buttons
  • Test keyboard navigation during development
  • Include skip links on all new pages

For Designers:
  • Ensure 7:1 contrast ratio for text
  • Don't use color alone to communicate information
  • Consider motion sensitivity in new animations
  • Test with zoom at 200%
  • Consider keyboard-only navigation

For QA/Testing:
  • Include a11y in test plans
  • Test with keyboard only (no mouse)
  • Test with screen reader
  • Document a11y issues separately
  • Verify fixes before closing tickets

For Management:
  • Schedule quarterly accessibility audits
  • Include accessibility in sprint planning
  • Budget time for a11y improvements
  • Train team on WCAG 2.1 basics
  • Set accessibility KPIs and track progress
